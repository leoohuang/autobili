# IDEA REPORT

## Topic

如何使用 Claude Code

## Platform And Format

- 平台：Bilibili
- 视频类型：实操型长视频
- 目标时长：12-16 分钟
- 目标受众：想把 AI 真正接入开发流程的中文开发者、独立开发者、AI 工具重度用户

## Why This Topic Matters

“AI 编程”这个话题已经不新了，但多数内容还停留在“它会写代码”“它比谁更强”这一层。真正让观众有留存的，不是再看一遍模型对比，而是有人把实际使用路径讲清楚：

1. Claude Code 到底适合什么场景。
2. 新手第一天应该怎么开始，不容易翻车。
3. 怎么把它从聊天工具，变成终端里的工作流工具。
4. 哪些能力值得依赖，哪些地方必须保留人工判断。

这个选题的现实张力在于：很多人已经听过 Claude Code，但并没有形成稳定使用习惯。他们不是缺“知道”，而是缺一条能马上照着做的路径。

## Candidate Angles

### Angle A

“Claude Code 是不是程序员新搭档”

- 优点：覆盖面广，容易吸引泛技术观众。
- 风险：太泛，容易变成功能罗列和空泛讨论。

### Angle B

“我用 Claude Code 重做一遍真实开发流程”

- 优点：可操作性强，容易建立信任。
- 风险：如果案例太复杂，观众会被细节淹没。

### Angle C

“新手怎样在第一周把 Claude Code 用顺手”

- 优点：天然有结构，从安装、提问、改代码、管上下文、做安全边界，到融入日常工作流，适合长视频。
- 风险：需要把“教学”拍得有节奏，否则会显得像文档念读。

## Recommended Direction

推荐 Angle C：

**把视频主线设定为：一个从没认真用过 Claude Code 的开发者，怎样在第一周里学会正确使用它。**

这个角度比“功能介绍”更强，原因有三点：

1. 它天然自带叙事推进，从陌生到熟练，观众更容易跟着走。
2. 它会自动筛掉不重要的功能，只保留真正会影响体验的关键动作。
3. 它能兼顾新手和中级用户。新手拿走入门路径，中级用户会关注工作流和边界管理。

## Core Promise

看完视频，观众至少能带走三件事：

1. 知道 Claude Code 最适合解决什么问题。
2. 知道该怎么提需求、怎么给上下文、怎么限制风险。
3. 知道怎样把它接进自己的日常开发流，而不是只把它当聊天机器人。

## Structural Risk

最大风险不是信息不够，而是内容太像“官方教程总结”，导致观众觉得自己看文档也能得到同样的信息。

规避方式：

- 每个章节都围绕一个真实使用问题展开，而不是围绕功能名展开。
- 每讲一个能力，都明确“为什么对新手重要”。
- 持续强调边界、误区和判断力，避免把工具吹成全自动开发神话。

## Benchmark Check

这个角度可以支撑长视频，因为它具备完整的中段内容：

- 开局可以建立共鸣：很多人知道工具，但不会真正用。
- 中段可以拆成 4 到 5 个实际使用节点。
- 结尾可以回到更高一级的问题：Claude Code 不是替你写完所有代码，而是改变你组织开发工作的方式。

如果只做“Claude Code 有哪些功能”，这条视频更像 3 到 5 分钟短视频，长视频会塌。
