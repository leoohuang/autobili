## 改进摘要

**健壮性提升** - 为 OpenAI API 调用添加超时配置

### 改了什么
在 `app/api/generate/route.ts` 中为两个 OpenAI API 调用添加了超时配置：
1. 分析视频结构的非流式请求：60 秒超时
2. 生成脚本的流式请求：120 秒超时

### 为什么改
原代码没有设置超时时间，如果 OpenAI API 响应缓慢或挂起，请求可能会无限期阻塞，导致：
- 用户体验差（页面一直显示"生成中..."）
- 服务器资源被长时间占用
- Vercel 等部署平台的 60 秒函数超时限制可能导致请求被强制终止

### 怎么改的
- 使用 OpenAI SDK v4 的 `RequestOptions` 参数（第二个参数）传递 `timeout` 选项
- 分析请求设置 60 秒超时（JSON 响应通常较快）
- 流式生成请求设置 120 秒超时（允许更长的生成时间）
- 超时后会抛出 `OpenAI.APIConnectionError`，已被现有的错误处理捕获
